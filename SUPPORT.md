# Support

## Getting help

- **Bugs / feature requests**: open a GitHub Issue.
- **Questions**: open a GitHub Issue with the label `question` (or use Discussions if enabled).
- **Security**: see [SECURITY.md](SECURITY.md). Please do not file public issues for vulnerabilities.

## What information to include

- `kwctl version`
- Cluster type + Kubernetes version
- Install method (Helm, `kwctl install`, manifests)
- Relevant logs (redact secrets):
  - `kwctl doctor`
  - `kwctl logs -f`

## Response expectations

This is a maintainer-led project. We’ll try to acknowledge issues within a few days, but response times can vary.
